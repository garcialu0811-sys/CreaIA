## 1. ¿Qué es un Diccionario? (Clave : Valor)

Hasta ahora, hemos guardado datos en Listas. Las listas organizan la información por su **posición** (índice 0, 1, 2...). Sin embargo, a veces es muy difícil recordar qué significa cada número. 

Los **Diccionarios** resuelven este problema utilizando un sistema de **"Clave y Valor"** (Key-Value). 

**La Analogía:**
Imaginen que están empacando para mudarse. En lugar de tener cajas transparentes numeradas (como una lista), tienen cajas de cartón cerradas. Para saber qué hay dentro sin abrirlas, le pegan una etiqueta con un nombre escrito con marcador. 
* La **Clave** es la etiqueta exterior de la caja (ej. "Ropa de invierno").
* El **Valor** es lo que hay dentro de la caja (ej. "Abrigos y bufandas").



### Sintaxis y Creación

A diferencia de las listas que usan corchetes `[ ]`, los diccionarios se crean usando **llaves `{ }`**. Separamos la clave y el valor con dos puntos `:`, y cada par de datos se separa con una coma `,`.

```python
# Un diccionario es ideal cuando queremos que cada dato tenga una etiqueta clara.
producto = {
    "nombre": "Arroz",      # Clave: "nombre", Valor: "Arroz"
    "precio": 1200.0,       # Clave: "precio", Valor: 1200.0
    "cantidad": 5           # Clave: "cantidad", Valor: 5
}
```

## 2. Acceder y Modificar Valores

### Acceder a un valor
Para ver qué hay dentro de una "caja", ya no usamos números (índices). Usamos exactamente el nombre de la etiqueta (la Clave) entre corchetes. 
*Nota: La clave debe escribirse exactamente igual, respetando mayúsculas y minúsculas.*

```python
producto = {
    "nombre": "Arroz",
    "precio": 1200.0,
    "cantidad": 5
}

# Extraemos información usando sus etiquetas
print("El producto es:", producto["nombre"])
print("El precio unitario es:", producto["precio"])

# Podemos hacer operaciones matemáticas directamente con los valores del diccionario
total = producto["cantidad"] * producto["precio"]
print(f"El valor total en inventario es: {total}")
```

### Modificar un valor existente
Funciona exactamente igual que en las listas, pero usando la clave en lugar del índice numérico.

```python
# En una lista se haría así: lista_edades[5] = 50
# En un diccionario se hace así:
producto["precio"] = 1500.0

# Ejemplo: Actualizar el inventario después de vender 1 unidad
producto["cantidad"] = producto["cantidad"] - 1

print("Producto actualizado:", producto)
```

### Agregar un nuevo elemento
Si llamamos a una Clave que no existe y le asignamos un valor, Python automáticamente crea esa nueva "etiqueta" y la añade al diccionario.

```python
# El diccionario original no tenía la clave "categoria". Se agregará al final.
producto["categoria"] = "Granos"
print(producto)
```

## 4. Recorrer un Diccionario (Ciclos)

Si queremos ver todo el contenido de un diccionario usando un ciclo `for`, tenemos dos formas principales de hacerlo:

### Forma 1: Recorrer solo las Claves (Por defecto)
Si pasamos el diccionario directamente al ciclo `for`, Python nos irá entregando las etiquetas (claves) una por una.

```python
print("--- Forma 1 ---")
for etiqueta in producto:
    # Usamos la etiqueta para extraer el valor en cada vuelta del ciclo
    print(f"Clave: {etiqueta} | Valor: {producto[etiqueta]}")
```

### Forma 2: Usar `.items()` (Recomendado)
El método `.items()` es sumamente útil porque nos entrega tanto la Clave como el Valor al mismo tiempo en cada vuelta del ciclo. Requiere dos variables temporales.

```python
print("--- Forma 2 ---")
for clave, valor in producto.items():
    print(f"Clave: {clave} | Valor: {valor}")
```

## 5. Comparación: Lista vs Diccionario

¿Por qué usar diccionarios si las listas ya guardan datos? Observen la diferencia en legibilidad:

**Con una Lista:**
```python
producto_lista = ["Arroz", 1200, 5]
# Si leemos este código en un mes, ¿recordaremos qué significa el índice 1?
print(producto_lista[1]) 
```

**Con un Diccionario:**
```python
producto_dicc = {"nombre": "Arroz", "precio": 1200, "cantidad": 5}
# El código se explica por sí mismo. Es evidente qué estamos imprimiendo.
print(producto_dicc["precio"])
```
**Conclusión:** Las listas son excelentes para colecciones de elementos similares (ej. una lista de notas, una lista de nombres). Los diccionarios son ideales para describir un objeto único con múltiples características distintas.

## 6. Errores Comunes (¡Cuidado con esto!)

1.  **Escribir mal la clave:** Buscar `producto["Nombre"]` cuando la clave real es `"nombre"`.
2.  **Acceder a una clave inexistente:** Pedir `producto["color"]` si esa etiqueta jamás fue creada en el diccionario. Esto detendrá el programa con un error `KeyError`.
3.  **Confundir sintaxis:** Intentar crear un diccionario con corchetes `[]` en lugar de llaves `{}`.
4.  **Olvidar las comillas en las claves:** Escribir `{nombre: "Arroz"}` en lugar de `{"nombre": "Arroz"}`. Sin las comillas, Python pensará que `nombre` es una variable.

## 7. Ejercicio Integrador: Lista de Diccionarios

**El Reto:**
Vamos a crear una lista gigante llamada `inventario`. Dentro de esta lista, cada elemento no será un simple número, sino un **diccionario completo** que represente un producto (con su nombre, precio y cantidad). 

El usuario podrá ingresar productos continuamente hasta que escriba la palabra "FIN". Al terminar, mostraremos todo el inventario de diferentes maneras.

```python
def mostrar_inventario(lista_productos):
    print("\n========= INVENTARIO FINAL =========")
    
    # Forma 1: Impresión línea por línea (Básico)
    print("\n--- Forma 1: Línea por línea ---")
    for i in range(len(lista_productos)):
        print(lista_productos[i]["nombre"])
        print(lista_productos[i]["precio"])
        print(lista_productos[i]["cantidad"])

    # Forma 2: Concatenación clásica convirtiendo a texto (str)
    print("\n--- Forma 2: Concatenación manual ---")
    for i in range(len(lista_productos)):
        # Necesitamos convertir los números (float, int) a texto (str) para unirlos con el signo +
        articulo = lista_productos[i]["nombre"] + " - $" + str(lista_productos[i]["precio"]) + " - " + str(lista_productos[i]["cantidad"]) + " unds."
        print(articulo)

    # Forma 3: Interpolación f-strings (¡La forma más recomendada y moderna!)
    # Permite inyectar las variables directamente dentro del texto sin convertir tipos manuales.
    print("\n--- Forma 3: Usando f-strings ---")
    for producto in lista_productos: # Usamos un for simplificado
        print(f"{producto['nombre']} - ${producto['precio']} - {producto['cantidad']} unds.")

    # Forma 4: Modificando el final del print (end="")
    print("\n--- Forma 4: Usando parámetro 'end' ---")
    for i in range(len(lista_productos)):
        print(lista_productos[i]["nombre"], end=" - $")
        print(lista_productos[i]["precio"], end=" - ")
        print(lista_productos[i]["cantidad"], "unds.")


# --- PROGRAMA PRINCIPAL ---

productos = [] # Iniciamos con una lista vacía que guardará nuestros diccionarios

print("Bienvenido al sistema de registro de productos.")

while True:
    # Usamos .upper() para que funcione si el usuario escribe "fin", "Fin" o "FIN"
    nombre = input("\nIngrese el nombre del producto (o 'FIN' para terminar): ").upper()
    
    if nombre == "FIN":
        mostrar_inventario(productos)
        break # Rompemos el ciclo infinito

    # Solicitamos los datos numéricos y los convertimos al tipo correcto
    precio = float(input("Ingrese el precio del producto: "))
    cantidad = int(input("Ingrese la cantidad en bodega: "))

    # Empacamos toda la información de esta vuelta del ciclo en un diccionario
    nuevo_producto = {
        "nombre": nombre,
        "precio": precio,
        "cantidad": cantidad
    }

    # Agregamos la "caja" (diccionario) completa a nuestro "estante" (lista principal)
    productos.append(nuevo_producto)
    print(f"¡Producto '{nombre}' agregado exitosamente!")
```
