## Convenciones de nombres
### *Regla 1: CamelCase(PascalCase) EXCLUSIVO para clases*
- no se permite espacios y esta estrictamente prhibido usar guiones bajos
- unica y exclusivamente para bautizar clases

* Ejemplos correctos: Cliente, Cliente Frecuente, VehiculoDeReparto, FacturaMensual.
* Ejemplos incorrectos: cliente, Cliente_Frecuente, vehiculoReparto

### *Regla 2: snake_case para Funciones y Variables*
- Si el mnombre esta compuesto por varias palabras, se unen utilizando un guion bajo (_)
- Para todo lo demas en el programa (variables, funciones, listas, diccionarios)

* Ejemplos correctos: calcular_pago(), cliente_frecuente(), vehiculo()
* Ejemplos incorrectos: CalcularPago(),
